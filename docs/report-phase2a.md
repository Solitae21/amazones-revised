# Phase 2-A 報告文案（社内レビュー用）

## 先方への報告文案

```
お疲れ様です。

前回いただいたフィードバックに基づき、トップページ（index.html）の
コーディング品質改善を実施いたしましたのでご確認をお願いいたします。

■ 改善内容

前回ご指摘いただいた「ユーティリティクラスの多用」「ユーティリティを
コンポーネントで上書きしている」点について、トップページのHTML構造を
改善いたしました。

具体的には、ユーティリティクラスの羅列をコンポーネントのSCSSに吸収し、
HTMLからスタイル指定を取り除いています。

【Before】
<h1 class="c-reveal u-fs-39 u-lh-normal u-font-outfit u-text-white u-fw-300">
<p class="u-fs-24 u-fw-500 u-font-zen u-text-black1 u-lh-200">

【After】
<h1 class="c-reveal p-top__hero-heading">
<p class="p-top__voice-title">

このように、各要素が意味のあるコンポーネント名を持ち、
スタイルはSCSS側で管理される構成に変更しております。

■ 改善結果（トップページ）

・ユーティリティクラスの使用行数: 184行 → 95行（48%減）
・タイポグラフィ系ユーティリティ: 約140箇所 → 24箇所（83%減）
・レスポンシブ対応の不具合修復: 10箇所
  （以前の修正でセレクタ不一致になっていた箇所を修正）

■ 見た目について

内部構造の改善のみとなりますので、ブラウザ上の表示は変更ございません。
PC / SP ともに表示崩れがないことを確認済みです。

■ 確認いただきたい点

1. この改善方向で問題ないか
2. 他ページ（残り22ページ）も同様に進めてよいか

ご確認のほどよろしくお願いいたします。
```
