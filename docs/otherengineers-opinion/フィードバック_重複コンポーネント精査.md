# クライアントフィードバック: 重複コンポーネントの精査

## フィードバック原文

> ・各ページごとにCSSやJSを作成いただくのは問題ないのですが、ページ特有のデザインではなくほぼ同一デザインのコンポーネントが存在している場合があるため、精査をお願いいたします。
> 例） p-reservation-flow__nav p-faq__nav ほぼ同一デザイン
> c-footer-banner c-announcement__section ほぼ同一デザイン

```scss
}
.p-{page}__links-item:hover {
  color: var(--base-lightBlue);
}
```

レスポンシブも同一:
- 1024px: `flex-wrap: wrap; gap: 1rem; margin-bottom: 6rem; padding-top: 2rem;`

→ **クラス名以外すべて同一。完全な重複。**

## 指摘例② `c-footer-banner` / `c-announcement__section` — CTA バナー

### 重複の全体像

| コンポーネント | SCSSファイル | 使用ページ数 |
|---------------|------------|------------|
| `c-footer-banner` | `_footer-banner.scss`（234行） | 5ページ（index, about, family, member, admission） |
| `c-announcement__section` | `_announcement.scss`（94行）+ `_global-responsive.scss` | 7ページ（visitor, payment, medical, storelist, personal, sportip, price） |

→ 合計 **12ページ** で使われている同目的のCTAセクション。

### CSSプロパティ比較

| プロパティ | c-footer-banner | c-announcement | 一致 |
|-----------|----------------|----------------|------|
| display | flex | flex | ✓ |
| flex-direction | column | column | ✓ |
| align-items | center | center | ✓ |
| justify-content | center | center | ✓ |
| background-color（内側） | rgba(255,255,255,0.9) | rgba(255,255,255,0.9) | ✓ |
| background-size | cover | cover | ✓ |
| position | relative | relative | ✓ |
| background-image（外側） | bg_banner.png | bg_camouflage01.webp | ✗ |
| max-width / width | max-width: 1000px | width: 67.5rem (1080px) | ✗（近似） |
| padding（内側） | 72px 16px 60px 16px | 5rem | ✗（近似） |
| gap | 53px | 2rem (32px) | ✗ |
| overflow | visible | hidden | ✗ |

**一致点:**
- Flexbox レイアウト（column, center, center）が完全一致
- 半透明白背景 `rgba(255,255,255,0.9)` が完全一致
- 背景画像 + cover の構成が同一
- CTA ボタンが同一（`c-btn--light-blue`）
- 「無料体験・見学予約」への誘導という目的が同一

**差異点:**
- 背景画像が異なる
- 幅制約の方法（max-width vs 固定width）
- パディング・ギャップの数値
- c-footer-banner はキャラクターステッカー付き（装飾要素）
- c-announcement はページによってテキストが異なる

### HTML構造比較

```html
<!-- c-footer-banner -->
<section class="c-footer-banner">
  <div class="c-footer-banner__container">
    <h2>まずはお気軽に無料体験・見学へ<b>GO!</b></h2>
    <p>説明テキスト...</p>
    <a href="reservation/" class="c-btn--light-blue">予約ボタン</a>
    <div class="c-footer-banner__stickers"><!-- キャラクター装飾 --></div>
  </div>
</section>

<!-- c-announcement__section -->
<section class="c-announcement__section">
  <div class="c-announcement c-maxWidth">
    <div class="c-announcement__title-container">
      <h2>見出しテキスト（ページにより異なる）</h2>
    </div>
    <div class="c-announcement__body"><p>説明テキスト...</p></div>
    <a><button class="c-btn--light-blue">予約ボタン</button></a>
  </div>
</section>
```

→ 構造は `見出し + 本文 + CTAボタン + (任意)装飾` で同一。

### 修正方針

統一コンポーネント `c-cta-banner` を新設し、差異はモディファイアで吸収:

```scss
.c-cta-banner { /* 共通: flex, column, center, 半透明白背景 */ }
.c-cta-banner--with-stickers { /* キャラクター装飾付きバリアント */ }
.c-cta-banner--wide { /* 幅広バリアント（旧 announcement） */ }
```

背景画像はページ別CSSまたはインラインで指定。

## 修正による影響評価

### 全体方針

ページ固有の `p-*` クラスを共通コンポーネント `c-*` に集約する。

### 影響範囲

| 修正対象 | 変更ファイル（SCSS） | 変更ファイル（HTML） | リスク |
|----------|-------------------|-------------------|--------|
| ① ナビゲーション | 新規1 + 既存6削除/縮小 | 6ページ | **中** |
| ② CTA バナー | 新規1 + 既存2統合 | 12ページ | **高**（全ページの主要CTA） |
| ③ セクションヘッダー | 新規1 + 既存8削除/縮小 | 8+ページ | **中** |
| ④ speak キャラクター | 新規1 + 既存7削除/縮小 | 7ページ | **中** |
| ⑤ ステップカード | 新規1 + 既存3削除/縮小 | 3ページ | **低** |
| ⑥ キャンペーンヘッダー | 新規1 + 既存4削除/縮小 | 4ページ | **低** |
| ⑦〜⑪ その他 | 各1新規 + 既存削除 | 各2-4ページ | **低** |

### リスクと対策

| リスク | 詳細 | 対策 |
|--------|------|------|
| 見た目崩れ | 微妙な数値差異（gap: 53px vs 2rem 等）を統一時に見落とす | 統一前に各ページのスクリーンショットを取得し、統一後に比較 |
| モディファイア設計ミス | 差異の吸収が不十分で一部ページだけ崩れる | 差異を全量リストアップし、モディファイア or ページ別上書きの判断を事前に行う |
| レスポンシブ崩れ | PC は合っていても SP で崩れる | PC / タブレット / SP の3ブレークポイントで全ページ確認 |
| CSS 詳細度の変化 | p-* → c-* に変更すると詳細度が変わる可能性 | 共通コンポーネント化後、既存の p-* による上書きが残っていないか grep で確認 |

### 推奨実施順

1. **まず①ナビゲーション**から着手（グループAの4ページはほぼコピペなので安全）
2. **次に⑨バナーオーバーレイ**（3行のルールなのでリスク極小、クイックウィン）
3. **次に③セクションヘッダー / ④ speak**（数は多いがパターンが安定）
4. **最後に② CTA バナー**（12ページに影響し、差異の吸収設計が必要なため慎重に）

## 確認事項（クライアント/デザイナーへ）

1. **CTA バナーの統一方針**: `c-footer-banner`（ステッカー付き）と `c-announcement`（テキスト中心）を完全統一するか、バリアント（モディファイア）で分けるか
2. **ナビゲーションの統一範囲**: FAQ / admission の微妙なレイアウト差異（中央寄せ vs 左寄せ等）はデザイン意図か、単なる実装差異か
3. **背景画像の差異**: `bg_banner.png` と `bg_camouflage01.webp` の使い分けはデザイン意図か
