# クライアントフィードバック: テキスト関連ユーティリティクラスの改善

## フィードバック原文

> ・u-font-zen u-fw-500 u-fs-16 などテキスト関連のユーティリティクラスに関しまして、全体を通して zen kaku gothic new weight Medium（500基準）のデザインに見えていましたが、ユーティリティクラスで吸収しないと難しいでしょうか。
> 特に u-font-zen に関してはhtml bodyにて var(--font-zen) 指定があるので、本来であれば指定不要だと想定しています。

---

## 調査結果サマリ

クライアントの指摘は妥当。現状の base スタイル設定が不十分なため、本来不要なユーティリティクラスが大量に使われている。

| ユーティリティ | 現在の使用数 | body/html での設定 | 削除可能な見込み |
|---------------|-------------|-------------------|----------------|
| `u-font-zen` | **673箇所** | `html, body { font-family: var(--font-zen); }` ✅ 設定済み | 大半が削除可能（一部例外あり） |
| `u-fw-500` | **2,233箇所** | `body { font-weight: ... }` ❌ **未設定** | body に 500 を追加すれば大半が削除可能 |

## 1. `u-font-zen` の詳細分析

### 現状

`_reset.scss` に以下の記述があり、全要素が Zen Kaku Gothic New を継承する:

```scss
html,
body {
  font-family: var(--font-zen);
}
```

`u-font-zen` ユーティリティの定義（`_typography.scss`）:

```scss
.u-font-zen {
  font-family: var(--font-zen);

```

→ body で既に設定済みのフォントを、ユーティリティで再指定しているだけ。**クライアントの指摘通り、大半は不要。**

## 2. `u-fw-500` の詳細分析

### 現状

**body に font-weight が設定されていない。**

```scss
// _reset.scss
html,
body {
  font-family: var(--font-zen);
  line-height: 100%;
  color: var(--text-black1);
  // ← font-weight の指定なし
}
```

ブラウザデフォルトの `font-weight: normal`（= 400）が適用されるため、デザインの基準値 500 にするには毎回 `u-fw-500` を付ける必要がある。**これが2,233箇所に膨れた原因。**

### 使用箇所（全2,233箇所 / 全 font-weight ユーティリティの74%）

| ファイル | u-fw-500 | u-fw-700 | u-fw-300 | u-fw-600 | u-fw-400 |
|----------|----------|----------|----------|----------|----------|
| index.html | 190 | 49 | 28 | 2 | 14 |
| information/index.html | 162 | 4 | 0 | 0 | 1 |
| faq/index.html | 134 | 69 | 0 | 0 | 1 |
| protein/index.html | 142 | 25 | 0 | 46 | 2 |
| member/index.html | 112 | 26 | 14 | 1 | 2 |
| admission/index.html | 109 | 16 | 14 | 1 | 3 |
| column/[slag]/index.html | 105 | 8 | 0 | 0 | 2 |
| storelist/[slag]/index.html | 98 | 24 | 14 | 2 | 3 |
| point_redemption/index.html | 94 | 43 | 0 | 0 | 4 |
| rules/index.html | 94 | 29 | 0 | 25 | 1 |
| about/index.html | 87 | 19 | 0 | 2 | 6 |
| family/index.html | 86 | 26 | 14 | 5 | 3 |
| medical/index.html | 84 | 16 | 14 | 17 | 1 |
| （その他10ページ） | 合計536 | 合計72 | 合計112 | 合計6 | 合計16 |
| **合計** | **2,233** | **426** | **196** | **107** | **58** |

### 修正方針

**`_reset.scss` に `font-weight: 500;` を追加する。**

```scss
html,
body {
  font-family: var(--font-zen);
  font-weight: 500;  // ← 追加
  line-height: 100%;
  color: var(--text-black1);
}
```

これにより、`u-fw-500` の大半が不要になる。

### 削除できないケース

`body { font-weight: 500; }` を設定しても、以下のケースでは `u-fw-500` が引き続き必要:

| ケース | 理由 | 該当例 |
|--------|------|--------|
| `<h1>`〜`<h6>` に付与されている場合 | ブラウザデフォルトが `bold`（700）のため、500 に下げるには明示指定が必要 | 見出し要素に `u-fw-500` が付いている箇所 |
| `<b>`, `<strong>` に付与されている場合 | 同上（ブラウザデフォルト 700） | 太字要素を 500 にしている箇所 |
| 親要素で別の font-weight が設定されている場合 | 継承値が 500 以外になる | `_storedetail-introduction.scss` の `font-weight: bold` 33箇所の子要素 |

### 補足: h1〜h6 のリセットも検討

現状の `_reset.scss` には h1〜h6 の font-weight リセットがない。デザイン基準が 500 であるなら、以下も追加を検討:

```scss
h1, h2, h3, h4, h5, h6 {
  font-weight: 500;
}
```

これにより、見出し要素上の `u-fw-500` も不要になる。ただし、見出しで意図的に 700 を使っている箇所（`u-fw-700` 付与）に影響はない（ユーティリティで明示上書きしているため）。

## 3. `u-fs-16` の詳細分析

### 現状

**body に font-size が設定されていない。**

ブラウザデフォルトの `16px` がそのまま適用されている。

```scss
// _reset.scss
html,
body {
  // ← font-size の指定なし（ブラウザデフォルト 16px）
}
```

### 使用箇所（全708箇所）

| ファイル | 箇所数 |
|----------|--------|
| faq/index.html | 118 |
| information/index.html | 75 |
| point_redemption/index.html | 51 |
| admission/index.html | 46 |
| family/index.html | 34 |
| rules/index.html | 31 |
| about/index.html | 29 |
| protein/index.html | 24 |
| medical/index.html | 21 |
| column/index.html | 19 |
| （その他13ページ） | 合計260 |

### 重要な発見: `u-fs-15` が最多使用

font-size ユーティリティの使用頻度:

| ユーティリティ | 使用数 | 備考 |
|---------------|--------|------|
| **u-fs-15** | **1,164** | 最多。デザイン基準サイズの可能性 |
| u-fs-16 | 708 | 2番目 |
| u-fs-14 | 379 | 3番目 |
| u-fs-18 | 258 | |
| u-fs-36 | 199 | |
| u-fs-24 | 136 | |
| u-fs-12 | 56 | |
| u-fs-100 | 24 | セクションヘッダー用 |

`u-fs-15`（15px）が最も多く使われている。デザインの本文基準サイズが 15px の可能性がある。

### 修正方針

`u-fs-16` は「ブラウザデフォルト 16px と同じ値を明示的に指定しているだけ」のケースが大半。

ただし、**`u-fs-15` が最多使用であることを考慮すると、デザインの基準本文サイズは 15px である可能性が高い**。この場合:

```scss
html,
body {
  font-size: 15px;  // ← デザイン基準サイズ
}
```

とすれば `u-fs-15` の1,164箇所が不要になるが、逆に `u-fs-16` は「基準と異なるサイズを指定する」ために引き続き必要になる。

**→ デザイン基準の本文サイズが 15px か 16px かをクライアント/デザイナーに確認する必要がある。**

### 削除できないケース（どちらの基準サイズでも共通）

| ケース | 理由 |
|--------|------|
| 親要素で別の font-size が設定されている場合 | 継承値が基準サイズと異なる |
| レスポンシブ切り替え（`u-fs-24 u-md-fs-16`） | PC/SP でサイズを変えている箇所（11箇所） |

---

## 修正方針まとめ

### Phase 1: ベーススタイルの整備（`_reset.scss` の修正）

```scss
html,
body {
  font-family: var(--font-zen);
  font-weight: 500;              // ← 追加
  font-size: ???px;              // ← 追加（15px or 16px — 要確認）
  line-height: 100%;
  color: var(--text-black1);
}
```

### Phase 2: 不要ユーティリティの削除（HTML側）

| 対象 | 削除見込み数 | 条件 |
|------|-------------|------|
| `u-font-zen` | 約488〜650箇所 | storelist/[slag] の一部を除き全削除 |
| `u-fw-500` | 約1,500〜2,000箇所 | h1-h6, b, strong 上を除き削除 |
| `u-fs-16` or `u-fs-15` | 約700〜1,100箇所 | 基準サイズと一致するものを削除 |

### Phase 3: 見出しリセットの追加（任意）

```scss
h1, h2, h3, h4, h5, h6 {
  font-weight: 500;  // ブラウザデフォルト bold→500 に統一
}
```

## 影響・リスク評価

| 項目 | リスク | 対策 |
|------|--------|------|
| `body { font-weight: 500 }` 追加 | **中** — 現在 400 で表示されている要素が 500 になる | `u-fw-400` が付いている要素（58箇所）は影響なし。付いていない要素で 400 前提のものがないか確認 |
| `body { font-size: 15px }` 追加 | **中** — ブラウザデフォルト 16px → 15px になり、ユーティリティ未指定の全テキストが 1px 小さくなる | 変更前後の全ページ目視確認が必要 |
| h1-h6 リセット追加 | **中** — `u-fw-700` なしで太字にしている見出しが 500 になる | 見出しの font-weight ユーティリティ付与状況を全量確認 |
| `u-font-zen` 削除 | **低** — body で既に設定済みのため見た目は変わらない | storelist/[slag] ページのみ慎重に対応 |
| `_storedetail-introduction.scss` の `html { font-family }` 問題 | **要確認** — デザイン意図の確認が必要 | 別件として切り出し、デザイナーに確認 |
