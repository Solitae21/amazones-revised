## Overview

Add `u-` prefix to all utility classes to resolve FLOCSS namespace conflicts.
Targets are SCSS definitions (9 files) + HTML references (23 files).

**Scope of this step (7a)**: `u-` prefix addition only. Naming convention changes (rem conversion, abbreviation) and utility inventory will be handled separately as future Steps 7b/7c.

**No JS file changes are required** (investigation confirmed JS does not reference utility classes).

### Note on Guideline Compliance

The coding guideline calls for abbreviated naming with rem units for Utility classes, such as `.u-fs1rem`, `.u-mt06rem`.
Current utilities (`.fs-15`, `.mt-10`, etc.) do not comply with this convention, but full naming unification involves very large work volume and risk, so it will be addressed incrementally.

| Step | Content | Purpose |
|------|------|------|
| **7a (this plan)** | `u-` prefix addition | Resolve FLOCSS namespace conflicts (especially `.p-*` padding vs `.p-*` Project) |
| 7b (future) | Delete unused utilities | Resolve bloat (1,094 classes) |
| 7c (future) | Full guideline compliance for naming/units | Unify to `.u-fs1rem` format, rem conversion |

## Risk Assessment

### High Risk

| Risk | Details | Countermeasure |
|--------|------|------|
| **Partial match false replacement** | `text-white` may partially match `c-hero__text-white` in HTML | Use word boundary (`\b`) regex for replacement in class attributes. Limit SCSS side to patterns following `.` at line start |
| **Bulk replacement of `relative` (188 locations)** | Most-used utility. Even one omission causes display breakage | Target only within HTML class attributes, limited by word boundary |
| **Bulk replacement of `font-outfit`/`font-zen` (1,666 total locations)** | Very numerous. Bulk replacement reliability is critical | `font-outfit` -> `u-font-outfit` has no partial match risk (no other classes contain this string) |
| **`text-*` ambiguity** | `text-lightBlue` is a utility, but `c-text-slider` is a component | Replace with exact match for `text-lightBlue`, `text-white`, etc., not `text-` alone |

### Medium Risk

| Risk | Details | Countermeasure |
|--------|------|------|
| **`m-{N}` false detection** | Many component class names in HTML contain `m-` | Do not bulk-replace `m-` alone; replace individually for defined values (m-0, m-4, m-5, ...) |
| **`.p-{N}` and FLOCSS `.p-` conflict** | Unused in HTML but SCSS definitions need changing | SCSS only. Limited to `\.p-(\d+)` pattern |
| **Mixed content in `_global-responsive.scss`** | Utilities (display-sp, etc.) and components (.c-maxWidth, etc.) are mixed | Target only utility classes, identifying and replacing line by line |

### Low Risk

| Risk | Details |
|--------|------|
| **Classes defined in SCSS only (unused in HTML)** | Many utilities are unused in HTML. Only SCSS-side changes, no impact |
| **CSS compilation** | Recompilation after SCSS changes auto-reflects |

## Processing Order

To prevent partial matches, process **longer class names -> shorter class names**.

### Phase 1: SCSS Utility File Replacement

Change class definitions (`.className`) to `.u-className` in each file.
Process each SCSS file as one replacement task.

**Processing order**:
1. `_responsive.scss` -- responsive variants (lg-*, sm-*)
2. `_space.scss` -- spacing
3. `_layout.scss` -- layout
4. `_font-size.scss` -- font related
5. `_color.scss` -- colors

7. `_display.scss` -- display
8. `_column.scss` -- columns

Bulk replace utility class names within class attributes.

**Category processing order**:

2. Font family (font-outfit, font-zen)
3. Text colors (text-lightBlue, text-lightPink, text-black1, etc.) -- longer names first

## Detailed Work Procedure

### SCSS Replacement Method

Bulk replace `.className` to `.u-className` in class definitions via sed/Edit for each SCSS file.

Example (_color.scss):
```
.bg-white -> .u-bg-white
.text-white -> .u-text-white
```

Example (_responsive.scss):
```
.lg-block -> .u-lg-block
.sm-fs-14 -> .u-sm-fs-14
```

### HTML Replacement Method

Replace using word boundaries within class attributes.

```
class="... text-white ..." -> class="... u-text-white ..."
class="... relative ..." -> class="... u-relative ..."
class="font-outfit fw-500 fs-15" -> class="u-font-outfit u-fw-500 u-fs-15"
```

**Caution**: The following patterns should **not be replaced**:
- `c-text-slider` -> The `text-slider` part is part of a component name
- `p-top__service-*` -> FLOCSS Project prefix
- `l-footer__*` -> FLOCSS Layout prefix

### Verification Method

1. After replacement, grep for old class names to confirm no residuals
2. Confirm SCSS compilation succeeds
3. Confirm appearance is unchanged on all pages in browser
