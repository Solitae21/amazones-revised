# 実行計画: 重複コンポーネント統合（FB3）

> コンテキスト: `docs/plans/feedback-improvement-context.md` §2-3
> 先方フィードバック: 「ページ特有のデザインではなくほぼ同一デザインのコンポーネントが存在している場合があるため、精査をお願いいたします」

---

## 先方指摘の原文

> 各ページごとにCSSやJSを作成いただくのは問題ないのですが、ページ特有のデザインではなくほぼ同一デザインのコンポーネントが存在している場合があるため、精査をお願いいたします。
> 例） `p-reservation-flow__nav` `p-faq__nav` ほぼ同一デザイン
> `c-footer-banner` `c-announcement__section` ほぼ同一デザイン

## 方針

### 原則

- **先方が名指しした2例（#1, #2）を最優先で対応する**
- #3（セクションヘッダー）は FB1 の Plan（チケット 1-5）で対応済みになる見込み → 確認のみ
- #4〜#11 は優先度順に対応。ただし★☆☆は工数対効果を見て先方に要否を確認する
- 統合は「共通コンポーネント（c-*）を1つ作り、差異は BEM Modifier で吸収」が基本方針
- FLOCSS の設計: 共通 → `c-*`（component）、差異が大きすぎて共通化できないもの → `p-*`（project）のまま残す

### 先方への確認が必要な事項

統合作業を始める前に、以下を先方に確認する:

1. **CTA バナー（#2）**: `c-footer-banner`（ステッカー付き）と `c-announcement__section`（ステッカーなし）の差異はデザイン意図か？ → 統一してよいか、バリアントとして残すか
2. **ナビゲーション（#1）**: FAQ / admission のレイアウト差異（max-width, flex-wrap 等）はデザイン意図か？
3. **★☆☆ のパターン（#7〜#11）**: 統合すべきか、現状維持でよいか

## チケット

### チケット F3-1: ページ内ナビゲーション統合

#### 何を
7箇所に分散しているページ内ナビゲーション（`p-*__links-nav` / `p-*__nav`）を共通コンポーネント `c-page-nav` に統合する。

#### なぜ
- 先方指摘例の1つ目
- グループA（reservation, price, payment, sportip）はクラス名以外すべて同一のコピペ
- storelist は類似だが若干差異あり
- FAQ / admission はレイアウト差異あり

#### 対象ファイル
- SCSS: `_reservation.scss`, `_price.scss`, `_payment.scss`, `_sportip.scss`, `_storelist.scss`, `_faq.scss`, `_admission.scss`
- HTML: 上記7ページの index.html
- 新規: `assets/scss/object/component/_page-nav.scss`（共通定義）

#### 受け入れ基準
- [ ] 共通コンポーネント `c-page-nav` / `c-page-nav__item` が作成されている
- [ ] グループA（4ページ）が共通コンポーネントのみで動作している
- [ ] storelist の差異が Modifier で表現されている
- [ ] FAQ / admission の差異が Modifier で表現されている（差異が大きすぎる場合は p-* のまま残し、理由を記録）
- [ ] 旧クラスの SCSS 定義が削除されている（またはコメントアウト）
- [ ] 全7ページでナビゲーションの見た目が変わっていない

---

### チケット F3-2: CTA バナー統合

#### 何を
`c-footer-banner`（5ページ）と `c-announcement__section`（7ページ）を1つの共通コンポーネントに統合する。

#### なぜ
- 先方指摘例の2つ目
- 共通点: Flexbox column/center、半透明白背景、背景画像 + cover、CTA ボタン
- 差異点: 背景画像、幅制約方法、パディング・ギャップ値、ステッカー有無

#### 前提
- **先方に統合方針の確認が取れてから着手**（ステッカー付き/なしの扱い）

#### 受け入れ基準
- [ ] 共通の CTA バナーコンポーネントが1つ存在する
- [ ] 差異（ステッカー、幅、パディング等）が Modifier で表現されている
- [ ] 旧コンポーネントの SCSS が統合・削除されている
- [ ] 全12ページで CTA バナーの見た目が変わっていない

---

### チケット F3-3: セクションヘッダー統合（確認のみ）

#### 何を
FB1 Plan のチケット 1-5（c-section-header 改善）で対応済みかを確認する。8箇所の `p-*__section-header` / `p-*__title-container` が `c-section-header` に統合されたかを検証。

#### なぜ
- 8箇所がページごとにコピペされている
- FB1 のチケット 1-5 で c-section-header にベーススタイルを追加するが、p-* 側の重複定義の削除まではスコープに含まれていない可能性がある

#### 受け入れ基準
- [ ] 8箇所の p-* セクションヘッダーが c-section-header の共通定義を使用している
- [ ] p-* 側の重複 SCSS 定義が削除されている
- [ ] 差異がある場合は Modifier で表現されている
- [ ] 全ページでセクションヘッダーの見た目が変わっていない

---

### チケット F3-4: キャラクター吹き出し統合

#### 何を
7箇所に分散している `p-*__speak-*` を共通コンポーネント `c-speak` に統合する。

#### なぜ
- 7つの SCSS ファイルにほぼ同一のスタイルがコピペされている

#### 受け入れ基準
- [ ] 共通コンポーネントが作成されている
- [ ] 7ファイルの p-* 定義が削除されている
- [ ] 全7ページで吹き出しの見た目が変わっていない

---

### チケット F3-5: バナーオーバーレイのユーティリティ化

#### 何を
3ファイルにコピペされている overlay スタイル（`position: absolute; inset: 0; background: rgba(0,0,0,0.4); z-index: 3; pointer-events: none`）をユーティリティクラス `u-overlay-dark` として定義する。

#### なぜ
- 同一の5行が3ファイルにコピペ
- コンポーネント化より**ユーティリティ化が適切**（単一の視覚効果、コンテキスト非依存）

#### 受け入れ基準
- [ ] ユーティリティクラスが定義されている
- [ ] 3ファイルの重複定義が削除されている
- [ ] 見た目が変わっていない
