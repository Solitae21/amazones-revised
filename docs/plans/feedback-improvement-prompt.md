# 次のチャットに貼り付けるプロンプト

以下をそのままコピーして新しいチャットに貼り付けてください。

---

`docs/plans/feedback-improvement-context.md` を読んでください。このファイルには、先方フィードバックへの対応として行うユーティリティクラス改善の調査結果・合意済み方針・技術的詳細がすべてまとめられています。

このコンテキストを踏まえて、**index.html（トップページ）を対象とした実行計画（Plan）** を作成してください。

## 前提条件

- デザイン基準フォントサイズ: **16px**
- デザイン基準フォントウェイト: **500**
- 見た目を絶対に崩さないこと（CLAUDE.md の最重要原則）
- 他ページへの影響を最小限に抑えること

## 実施順序（合意済み）

```
Step 0: global.scss の読み込み順を FLOCSS 正規順に修正
Step 1: ベーススタイル整備（_reset.scss — body + h1-h6）
Step 2: コンポーネント SCSS にユーティリティを吸収（追加のみ）
Step 3: index.html の冗長ユーティリティを削除
Step 4: footer-banner 3重定義の修正
Step 5: コンパイル・最終検証
```

## Plan に含めてほしい内容

各 Step について以下を明記してください:

1. **変更対象ファイル** — パスと行番号
2. **変更前のコード** — 現状の SCSS/HTML（該当箇所を実際に読んで確認すること）
3. **変更後のコード** — 具体的な修正内容
4. **影響範囲** — このステップで他ページに影響が出るかどうか
5. **検証方法** — 変更後にどう確認するか

特に Step 2（コンポーネント吸収）と Step 3（ユーティリティ削除）は、対象コンポーネントごとに個別に記載してください。コンテキストファイルのセクション5「コンポーネント SCSS の現状」に7つのコンポーネントの吸収方針が記載されています。

## 注意事項

- コンテキストファイルの内容を信頼しつつも、**実際のファイルを読んで現状を確認してから** Plan を作成してください（行番号やコードが変わっている可能性があるため）
- Plan 完成後は `docs/plans/step9-utility-improvement-plan.md` に保存してください
- Plan が完成したら内容を提示して承認を求めてください
