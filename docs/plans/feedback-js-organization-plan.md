# 実行計画: global.js / common.js 整理（FB4）

> コンテキスト: `docs/plans/feedback-improvement-context.md` §3-2
> 先方フィードバック: 「global.js common.js の使い分けについて、使い分けの判断をご提示いただけますでしょうか」

---

## 先方指摘の原文

> global.js common.jsの使い分けについて以前言及していましたが、使い分けの判断をご提示いただけますでしょうか。（global common はほぼ同義だと認識しております）

---

## 現状

| ファイル | サイズ | 役割 | 読み込み |
|---------|--------|------|---------|
| `common.js` | 3.2KB | 汎用関数3つ（smoothScroll, carousel, modal） | 全ページ |
| `global.js` | 35.9KB | ページ共通UI（header, floating menu, animation, slider） | 全ページ |

### 問題点

1. **名前がほぼ同義**: 「global」と「common」は同じ意味。使い分けの基準が不明
2. **スムーズスクロールが二重実装**: common.js の `initSmoothScroll` と global.js の `smoothScrollTo` が別実装で共存
3. **両方とも全ページで読み込み**: 分割の意味がない

---

## 方針

### 先方が求めているもの

先方は**コード修正ではなく「使い分けの判断基準の提示」**を求めている。つまり:

1. なぜ2ファイルに分かれているのかの説明
2. 今後どういう基準で使い分けるのかの方針

### 2つの選択肢

#### A案: common.js を global.js に統合する（推奨）

```
Before:
  common.js (3.2KB) — 汎用関数
  global.js (35.9KB) — ページ共通UI

After:
  global.js (39KB) — 全ページ共通の機能すべて
  各ページJS — ページ固有の機能（top.js, about.js 等）
```

理由:
- 両方とも全ページで読み込んでいる → 分ける意味がない
- 名前の混乱が解消される
- 使い分け基準が明確: **global.js = 全ページ共通、{page}.js = ページ固有**

#### B案: 役割で再定義する

```
Before:
  common.js — 汎用関数
  global.js — ページ共通UI

After:
  lib.js — 再利用可能なユーティリティ関数（initSmoothScroll, initModal 等）
  global.js — ページ共通UI（header, footer, animation）
  各ページJS — ページ固有
```

理由:
- 「ライブラリ関数」と「UIロジック」は概念的に異なる
- ただし 3.2KB のファイルのために別ファイルを維持する価値があるか疑問

### 推奨: A案

common.js の 3 関数は global.js に統合し、スムーズスクロールの重複を解消する。
ファイル構成を「global.js + ページ別JS」の2層に簡素化する。

## チケット

### チケット F4-1: 使い分け方針の整理と先方への提示

#### 何を
global.js / common.js の使い分け方針を整理し、先方に提示する資料を作成する。

#### なぜ
- 先方が「使い分けの判断」の提示を求めている
- コード修正の前に方針の合意が必要

#### 受け入れ基準
- [ ] 現状の2ファイルの役割・サイズ・重複点が整理されている
- [ ] 今後の使い分け方針（A案 or B案）が明記されている
- [ ] 先方に提示できる形式になっている

---

### チケット F4-2: スムーズスクロール重複の解消

#### 何を
common.js の `initSmoothScroll` と global.js の `smoothScrollTo` の重複を解消する。

#### なぜ
- 同じ機能が2つの異なる実装で存在

#### 前提
- **先方に方針の合意を取ってから着手**（チケット F4-1 完了後）

#### 受け入れ基準
- [ ] 2つの実装の呼び出し箇所・引数・挙動の差異が調査されている
- [ ] スムーズスクロールが1実装に統一されている
- [ ] 全ページでスクロール動作が変わっていない
- [ ] 削除した実装の呼び出し箇所がすべて新実装に移行されている

---

### チケット F4-3: common.js の global.js への統合（A案の場合）

#### 何を
common.js の残り2関数（initMobileCarousel, initModal）を global.js に移動し、common.js を削除する。

#### なぜ
- A案で合意した場合、ファイル構成を簡素化する
- 全ページの HTML から `<script src="common.js">` を削除する

#### 受け入れ基準
- [ ] common.js の全関数が global.js に移動されている
- [ ] 全 HTML ファイルから common.js の読み込みが削除されている
- [ ] common.js ファイルが削除されている
- [ ] 全ページでカルーセル・モーダルの動作が変わっていない

